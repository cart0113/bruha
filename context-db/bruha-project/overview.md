# bruha (project notes)

This repo ships a docsify extension theme. A few things worth knowing before
making changes.

- **`docs/bruha.json` is the single source of truth.** `docs/index.html`,
  `docs/themes/bruha-config.js`, and every `_sidebar.md` are generated by
  `docs/bin/build.sh` — never edit those by hand. If your change requires
  touching a generated file, the fix belongs upstream in the build script or
  the JSON config.
- **`build.sh` is the API.** It runs in under 100 ms and is meant to be a
  pre-commit hook. Anything that adds friction (extra steps, slow paths,
  manual file edits) defeats the design goal of "agents just edit markdown
  and commit."
- **Distribution is copy-based, not packaged.** Downstream sites copy
  `docs/themes/` and `docs/bin/` into their own repo. There is no version
  pinning or update channel — breaking changes need to be obvious.
- **Python is stdlib only.** No external packages. Use `python3` and
  `ruff-main`.
- **Formatting is delegated.** Prettier / ruff run from the GIT_STANDARDS
  pre-commit hook; bruha itself does not format files.

For everything else — what bruha is, the feature catalog, install
instructions — see the README or the docs site under `docs/src/`.
