# Workflow

## Quick Reference

1. All content lives under `docs/src/` (configurable via `content_folder` in
   `docs/bruha.yaml`).
2. A directory contains **either** files **or** sub-directories — never both.
3. Use `_order.md` to control sidebar order (optional — default is
   alphabetical).
4. Start every `.md` page with a `# heading` (becomes the sidebar label).
5. After any content change, run `docs/bin/build.sh` to regenerate the sidebar.

## Adding a New Page

1. Pick the folder under `docs/src/` where the page belongs. The folder must
   already contain only files (no sub-folders).
2. Create a `.md` file. Start it with a `# heading`.
3. If the folder has an `_order.md` and you want the page in a specific
   position, add the file stem (without `.md`) to `_order.md`. Otherwise it
   appends alphabetically.
4. Run `docs/bin/build.sh`.

## Adding a New Section (Folder)

1. Create the folder under `docs/src/`.
2. The parent directory must contain only sub-folders (no loose `.md` files).
3. Add pages inside the new folder.
4. Optionally add `_order.md` in the new folder and/or update the parent's
   `_order.md`.
5. Run `docs/bin/build.sh`.

## Files You Must Never Hand-Edit

These are generated by `docs/bin/build.sh` and will be overwritten:

- `docs/_sidebar.md` — generated from filesystem + `_order.md` files
- `docs/themes/bruha-config.js` — generated from `docs/bruha.yaml`
- `docs/index.html` — generated from `docs/bruha.yaml`

## Links and Images

- Links are relative to `docs/`, not to the current file. Always include the
  `src/` prefix: `[label](src/section/page.md)`.
- Images go in the same folder or in `src/_assets/`. Reference from `docs/`
  root: `![alt](src/section/_assets/img.png)`.

## Callout Boxes

```markdown
> [!note] Blue info box.

> [!tip] Green tip box.

> [!important] Amber box.

> [!warning] Red box.
```

## Build Step

Always run after adding, removing, or renaming content:

```bash
docs/bin/build.sh
```

This rebuilds `_sidebar.md`, `bruha-config.js`, and `index.html`, then runs
prettier on JS, CSS, and MD files.
