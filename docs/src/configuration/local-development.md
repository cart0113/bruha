# Local Development

## Installing bruha in a Project

bruha is installed by **copying** files — there is no CDN or package manager. Do
not symlink theme files to external repos; symlinks break on GitHub Pages and
create fragile cross-repo dependencies.

Copy these directories from GIT_BRUHA into your project:

```
# from GIT_BRUHA, copy into your project:
docs/themes/    →  your-project/docs/themes/
docs/bin/       →  your-project/docs/bin/
```

Then create your own `docs/index.html`, `docs/bruha.yaml`, and `docs/src/`
content. The result:

```
your-project/
└── docs/
    ├── index.html       ← docsify entry point (model after GIT_BRUHA's)
    ├── bruha.yaml       ← your config overrides
    ├── .nojekyll        ← required for GitHub Pages
    ├── assets/          ← your icons (favicon, og-image)
    ├── bin/             ← copied from GIT_BRUHA
    │   ├── build.sh
    │   ├── serve.sh
    │   └── format.sh
    ├── src/             ← your markdown content
    │   ├── _order
    │   ├── _sidebar.md  ← generated by build.sh
    │   └── [folders]/
    └── themes/          ← copied from GIT_BRUHA
        ├── bruha-config.js  ← generated by build.sh
        ├── code-one.css
        ├── color-themes.css
        ├── bruha.css
        ├── mobile-nav.css
        ├── code-enhancements.js
        ├── collapsible-folders.js
        ├── mobile-nav.js
        ├── sidebar-indicator.js
        ├── theme-controls.js
        ├── top-nav.js
        └── folder-redirect.js
```

### Updating

To pick up bruha updates, re-copy `docs/themes/` and `docs/bin/` from GIT_BRUHA.
The generated files (`_sidebar.md`, `bruha-config.js`) will be rebuilt by
`build.sh`.

## Scripts

### serve.sh

Build and start a local docsify server:

```bash
docs/bin/serve.sh              # port 3000 (default)
docs/bin/serve.sh --port 4000  # custom port
```

### build.sh

Generate `_sidebar.md` and `bruha-config.js` without starting a server:

```bash
docs/bin/build.sh
```

The build script automatically finds the bruha Python source — first checking if
it's in the current repo (`src/bruha/`), then looking for a sibling `GIT_BRUHA`
repo (`../GIT_BRUHA/src/bruha/`).

### format.sh

Run prettier on all docs content and theme files:

```bash
docs/bin/format.sh
```

## Requirements

- `python-main` with PyYAML installed
- `docsify-cli` (`npm install -g docsify-cli`)
- `prettier` (via npx or installed locally)
